What Brief 2 delivered

Completed games now tell a deeper story without weakening the original bracket experience.

The user experience

  • Hover a completed match header for compact FIFA facts.
  • Hover a country row for country history, not repeated tournament placement.
  • Click, tap, Enter, or Space to open complete match details.
  • Use one arrow to expand match details, then a separate arrow to expand only the portrait.
  • See a first-use toast, plus one delayed reminder after a fast dismissal.
  • Use the same interaction in Mirrored and Sideways without adding cramped Sideways cards.
ELI5

The bracket became a storybook

The score card is the book cover. Hovering shows the short summary. Clicking opens the full page. The animated portrait is a picture inside that page. It stays asleep until you clearly ask to see it.

  • Goals, cards, venue, attendance, referee, round, score, and source.
  • 25 reviewed external portrait identities with exact slugs and IDs.
  • Future FIFA facts generate during the existing result sync.
  • Future external slugs remain manual until author permission exists.

What

Defined a strict JSON contract and froze exact FIFA calendar and M97 live payloads.

Why

Live APIs change. Frozen inputs let the parser prove the same result every time.

Learning moment

A reliable feature begins with a stable contract, not a component. Tests become much easier once inputs and outputs are explicit.

Core architecture observation

This is a static no-build application. Python prepares shared result data. Vanilla JavaScript renders each person's local bracket. Optional match details are loaded from same-origin JSON and never block the core dashboard.

ELI5

Kitchen and dining room

Python is the kitchen that prepares clean ingredients. JSON is the serving tray. JavaScript is the dining room that arranges the meal for each visitor. If one optional side dish is missing, dinner still works.

Portrait identity has four locks

1. Team pair

Both reviewed teams must equal the FIFA participants.

2. Oriented score

The score follows the mapping's team order and treats shootouts separately.

3. FIFA date

The canonical kickoff date must agree, including reviewed midnight rollover cases.

4. Stage

Round of 16, Quarter-final, and other labels must normalize to the same stage.

Allowed host

Every URL starts with exactly https://wc26.bogachev.fr.

Allowed slug

Only a reviewed lowercase abc-def slug is accepted.

ELI5

The four-key safe

A portrait opens only when four keys fit: teams, score, date, and round. A similar-looking match with the wrong key stays locked.

ELI5

One television, three sizes

The hover card, match dialog, and full visual do not create three televisions. They move and resize one television, so the video does not restart or count twice.

Learning moments to reuse

These ideas apply well beyond a World Cup bracket.

1. Contract before component

Lesson: Decide the data shape and failure rules before drawing the UI. This keeps parsing, rendering, and tests aligned.

2. Stable IDs beat names

Lesson: Names and three-letter codes vary. Preserve source IDs, then verify names as evidence instead of treating labels as identity.

3. Fail closed at joins

Lesson: External data should disappear on uncertainty. A missing portrait is better than a confident wrong portrait.

4. Intent before cost

Lesson: Delay external work until user intent is clear. This improves privacy, performance, and accidental analytics counts.

5. One object, many presentations

Lesson: Reuse one iframe and resize it. Preserving state prevents duplicate requests and restarts.

6. Parent and child mean different things

Lesson: Match hover belongs to the outer shell. Country hover belongs to the row. Event delegation should respect that meaning.

7. Optional means optional

Lesson: Scores and picks must survive a detail-file outage. Optional features should never become hidden hard dependencies.

8. Freeze moving inputs

Lesson: A live feed is a moving target. Frozen fixtures make parser tests repeatable and make regressions explainable.

9. Permission is architecture

Lesson: Legal and author approval gates belong in the design. They affect automation, deployment, attribution, and fallback behavior.

The shortest summary

Trust local facts. Verify external identity. Load only on intent. Clean up every time.

Those four rules explain most of Brief 2.
