Card contract

Each game becomes one visible object

One outer card holds the match header and both team rows. The same card component renders Actual path and My picks with different occupants and states.

  1. 1Header: match code, round or date, and completed or upcoming state.
  2. 2Team rows: bundled flag, country name, score, or a clear Winner MXX placeholder.
  3. 3Status: winner, loser, correct pick, missed pick, actual replacement, pending, and champion treatment.
  4. 4Access: useful label, visible focus, readable names, and no color-only meaning.

Required states

Completed

Final score, decisive winner, subdued loser, and any existing pick symbol.

Upcoming

Known teams or feeder placeholders, kickoff date, and a neutral pending state.

Shootout

Match score and penalty outcome remain unambiguous without crowding the card.

Flow and meaning

Connect games by feeder relationships

The current connector code searches for the same team in adjacent rounds. The rebuild connects child matches to parent matches using explicit topology edges.

  • Each feeder draws one elbow line into its parent.
  • Left-side lines travel right toward the Final.
  • Right-side lines travel left toward the Final.
  • M101 and M102 meet at M104.
  • The SVG stays behind cards and ignores pointer input.
  • Lines redraw after resize, font loading, theme changes, section changes, and bracket view changes.

Legend stays in place

Keep the existing How to read this bracket block. It must still explain a won pick, eliminated pick, correct path line, pending line, actual replacement, actual path line, advancing team, and champion pick. Update wording only where match-level lines require clearer language.

Center stage

A World Cup-inspired sculpture supports the Final

The model will use familiar high-level cues, but its styling comes from the dashboard: faceted geometry, matte token colors, compact proportions, and controlled lighting.

  • Faceted globe, two clean V supports, short bridge, and compact hexagonal plinth
  • Dashboard --gold, --panel, blue, and teal tokens
  • Flat-shaded matte surfaces with no chrome or image textures
  • Orthographic framing and restrained key plus rim lighting
  • No flowing ribbons, organic figures, official twisting stem, inscriptions, logos, or FIFA branding
  • No downloaded, traced, scanned, reconstructed, or official mesh
  • Pinned self-hosted Three.js ES module
  • One full vertical-axis rotation every 36 seconds
  • Subtle three-degree cursor-follow tilt
  • Click-drag rotation with grab and grabbing cursors
  • Touch yaw that preserves vertical page scrolling
  • Arrow key rotation, Home reset, and Space pause or resume
  • No inertia, bobbing, tilt loop, particles, bloom, or wheel zoom
  • Lazy initialization when the bracket enters view
  • Pause offscreen and while the tab is hidden
  • Reduced motion disables automatic and cursor-follow movement
  • Local SVG fallback for WebGL failure and phones
  • Pointer capture only while actively dragging

Design rule: make it look like a native dashboard illustration, not a separate art installation. Review it beside the Final card. Scale down or simplify anything that becomes louder than the tournament information.

Motion: one very slow 360-degree turn, subtle hover tilt, and direct drag. Auto-rotation resumes after release. No inertia or extra effects.

This design boundary reduces copying risk but does not provide legal clearance. Rights-holder permission or qualified legal review remains the path to certainty when public use requires it.

Confirmed choice: keep this distinct inspired design rather than reproduce the official trophy. The project's noncommercial status does not change the build boundary.

Responsive and accessible

Preserve every game at every review width

Phone

Show the whole tree as a miniature overview, then all 31 readable cards in a vertical round-by-round flow.

No side scrolling: no bracket view uses horizontal overflow, hidden scrollbars, drag-to-pan, or a global scale transform. Actual path and My picks update every responsive representation together.

  • Status never relies on color alone.
  • Decorative lines stay hidden from assistive technology.

  • Keyboard focus stays visible.

  • Three-letter visual team codes retain full names in accessible labels.

Definition of done

Sixteen checks define the first milestone

  1. Both bracket views contain exactly 31 unique game cards.
  2. The map contains exactly 30 feeder connections.
  3. M101 and M102 feed M104 at the center.
  4. The expected eight-game left and right Round-of-32 branches are correct.
  5. No match card is clipped or overlaps another at review widths.
  6. Flags, names, scores, dates, and placeholders remain legible.
  7. Existing pick and actual status meanings remain intact.
  8. The legend explains every visible state.
  9. The World Cup-inspired sculpture is original, self-hosted, gently animated, visually distinct from the official trophy, and non-essential.
  10. Reduced motion and WebGL failure show a clear static cup.
  11. Both views switch without layout jumps or stale connectors.
  12. No target width produces bracket horizontal overflow or side panning.
  13. The phone overview has all 31 nodes and 30 edges, and its vertical list has all 31 readable cards.
  14. Existing scoring, builder, share, and parse behavior remains unchanged.
  15. No data hover, FotMob, FIFA detail, or Bogachev integration is included.
  16. Local screenshots receive approval before the next phase starts.
